What Is Fine-Tuning? / 10 of 10

Alignment
& RLHF

Why SFT isn't enough:
An SFT model can be smart but unhelpful, biased, or toxic. We need to align it with human values.

  • Alignment: Helpful, Honest, Harmless (HHH)
  • Bias Reduction: Penalize toxic or skewed outputs

Popular Methods:

RLHF DPO (Direct Preference Optimization)
Prompt: "Tell me a joke about engineers"
"Engineers are just people who don't know how to talk to others..."
Rejected (Biased)
"An optimist says the glass is half full. The engineer says it's twice as big as it needs to be."
Preferred (Harmless)
Model weights update to favor Response B

"RLHF uses human preferences to steer the model towards helpful behaviors"

What Is Fine-Tuning? / 5 of 10

Full Parameter
Fine-Tuning

Update ALL Weights

During Full Fine-Tuning, the training data generates loss gradients that backpropagate through the entire network.

For a 1B model, this means computing and updating every single one of the 1,000,000,000 parameters across all layers simultaneously.

"Full fine-tuning updates everything = best results but massive cost"

What Is Fine-Tuning? / 7 of 10

The Solution:
PEFT

Parameter-Efficient Fine-Tuning

  • Freeze original weights (no updates!)
  • Track only the CHANGE (delta Δ)
  • Add delta to frozen weights during inference
W_finetuned = W_pretrained + ΔW

W_pretrained: Frozen (never changes)
ΔW: Trainable adapter (only this updates!)

Wait... if ΔW is the same size as W, how does this save memory?

"PEFT preserves original weights, only learns the change"

What Is Fine-Tuning? / 6 of 10

Problems with Full FT

$ Hardware & Cost

Updating all parameters requires storing the entire model in GPU memory.

Fine-tuning Llama-2 70B (Full Params):

  • GPU Memory: 280GB+ VRAM (need 4× A100s!)
  • Training Time: 200+ hours
  • Cost: $8,000–15,000 per run
  • Storage: 280GB per checkpoint
🎯 Fun Fact: GPT-4 trained on ~25,000 A100 GPUs for over 90 days — racking up a bill close to $100 million. 🔥 💸 🔥
⚠️ Catastrophic Forgetting

Training heavily on YOUR data → Model forgets general knowledge!

Example:

Before

"Translate English to French" ✓ Works

After fine-tuning on Medical Data

"Translate English to French" × Broken!

Understanding
Rank

What is "Rank"?
The number of independent dimensions in a matrix.

Full Rank Update (Rank = 3):
A standard 3×3 matrix requires learning 9 parameters.

Low-Rank Update (Rank = 1):
We can split it into a 3×1 and a 1×3 matrix.
Only 6 parameters to train, but produces 9 values when multiplied!

Full Rank: 9 params

What Is Fine-Tuning? / 9 of 10

LoRA
in Action

Original Weight Matrix: W
[1024 × 1024] = 1,048,576 params

Full Fine-Tuning:
Train all 1,048,576 parameters → Expensive!

LoRA Approach:
1. Freeze W (don't train!)
2. Add Low-Rank Adapter: ΔW = A × B

  • A: [1024 × 8] = 8,192 params
  • B: [8 × 1024] = 8,192 params
  • Total: 16,384 trainable params
Reduction: 64× fewer parameters! 🎉

During Inference:

Output = (W + A×B) × Input
↑
frozen ↑
trained

"LoRA trains tiny matrices A and B, gets full matrix performance"

How It's Done

The Full Pipeline

1
Load Model

Pick a pre-trained base (Llama, Mistral, Gemma…) from HuggingFace

2
Prepare Data

Format into prompt/response pairs — tokenize and batch

3
Set Hyperparams

LR, epochs, LoRA rank & alpha — tune before training starts

4
Train

Run the fine-tuning loop — loss decreases, ΔW adapters update

5
Merge Weights

Fold ΔW into base weights — single standalone model, no adapter overhead

6
Inference

Serve the fine-tuned model — faster, cheaper, task-specialized responses
